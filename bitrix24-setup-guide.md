# Пошаговая настройка автоматизации в Битрикс24 — VerdeTech

> Инструкция: куда кликать, что вставлять, в каком порядке
> Привязка к шаблонам: bitrix24-email-templates.md
> Привязка к триггерам: bitrix24-trigger-map.md

---

## Порядок настройки

```
1. Создать пользовательские поля (5 мин)
2. Создать email-шаблоны (30 мин)
3. Настроить роботов в воронке лидов (20 мин)
4. Настроить роботов в воронке сделок (30 мин)
5. Тестирование на тестовом лиде (10 мин)
```

---

## Шаг 1. Создать пользовательские поля в сделке

Эти поля нужны для подстановки переменных в шаблоны писем.

### 1.1. Откройте любую сделку

`CRM` → `Сделки` → откройте любую карточку сделки

### 1.2. Нажмите "Создать поле"

В карточке сделки → прокрутите вниз → кнопка **«Создать поле»** (или значок шестерёнки → «Настроить поля»)

### 1.3. Создайте поля по таблице

| Название поля | Тип | Обязательное | Зачем |
|---|---|---|---|
| Образцы отправлены | Да/Нет | Нет | Триггер для письма D2 |
| Дата отправки образцов | Дата | Нет | Для отслеживания |
| Трек-номер | Строка | Нет | Подставляется в D2, D8 |
| Дата доставки | Дата | Нет | Подставляется в D2, D8 |
| Список образцов | Текст | Нет | Подставляется в D2 |
| Номер счёта | Строка | Нет | Подставляется в D5-D7, D9 |
| Сумма счёта | Число | Нет | Подставляется в D5-D7 |
| Дата оплаты | Дата | Нет | Подставляется в D5-D6 |
| Номер заказа | Строка | Нет | Подставляется в D8 |
| Способ доставки | Список | Нет | Подставляется в D8 |

> **Для каждого поля:** Тип → выберите из выпадающего списка → Название → вставьте из таблицы → Сохранить.

> **Важно:** Если у вас уже есть интеграция с 1С — поля «Номер счёта», «Сумма» и «Дата оплаты» могут заполняться автоматически. Уточните у администратора.

## Шаг 2. Создать email-шаблоны

### 2.1. Откройте раздел шаблонов

`CRM` → `Настройки` (шестерёнка внизу слева) → `Автоматизация` → `Почтовые шаблоны`

Или: `Маркетинг` → `Рассылки` → `Шаблоны` (зависит от версии Б24)

### 2.2. Создайте шаблон для каждого письма

Нажмите **«Создать шаблон»** и заполните:

**Для каждого шаблона:**

1. **Название шаблона** — вставьте ID из нашего файла (например: `L1 — Автоответ при создании лида`)
2. **Тема письма** — скопируйте из bitrix24-email-templates.md
3. **Текст письма** — скопируйте тело письма

### 2.3. Замените переменные на макросы Битрикс24

При копировании текста **замените наши переменные** на макросы Б24:

**Для шаблонов ЛИДОВ (L1-L6):**

| В шаблоне | В Битрикс24 | Как вставить |
|---|---|---|
| {{Имя}} | `#LEAD.NAME#` | Кнопка `#` → Лид → Имя |
| {{Компания}} | `#LEAD.COMPANY_TITLE#` | Кнопка `#` → Лид → Компания |
| {{Имя менеджера}} | `#LEAD.ASSIGNED_BY.NAME#` | Кнопка `#` → Ответственный → Имя |
| {{Телефон менеджера}} | `#LEAD.ASSIGNED_BY.WORK_PHONE#` | Кнопка `#` → Ответственный → Рабочий телефон |
| {{Телефон}} | `#LEAD.PHONE#` | Кнопка `#` → Лид → Телефон |
| {{Email}} | `#LEAD.EMAIL#` | Кнопка `#` → Лид → Email |
| {{Источник лида}} | `#LEAD.SOURCE#` | Кнопка `#` → Лид → Источник |

**Для шаблонов СДЕЛОК (D1-D15):**

| В шаблоне | В Битрикс24 | Как вставить |
|---|---|---|
| {{Имя}} | `#CONTACT.NAME#` | Кнопка `#` → Контакт → Имя |
| {{Компания}} | `#COMPANY.TITLE#` | Кнопка `#` → Компания → Название |
| {{Имя менеджера}} | `#DEAL.ASSIGNED_BY.NAME#` | Кнопка `#` → Ответственный → Имя |
| {{Телефон менеджера}} | `#DEAL.ASSIGNED_BY.WORK_PHONE#` | Кнопка `#` → Ответственный → Телефон |
| {{Стадия}} | `#DEAL.STAGE#` | Кнопка `#` → Сделка → Стадия |
| {{Сумма}} | `#DEAL.OPPORTUNITY#` | Кнопка `#` → Сделка → Сумма |

> **Для пользовательских полей** (трек-номер, номер счёта и т.д.): нажмите кнопку `#` → Сделка → прокрутите до блока пользовательских полей → выберите нужное.

> **Подсказка:** Точные названия макросов могут отличаться в вашей версии Б24. В редакторе шаблона всегда есть кнопка `#` — через неё выбирайте поля из списка, не вбивайте макросы вручную.

### 2.4. Список шаблонов для создания

Создайте **17 шаблонов** (4 внутренних уведомления L2, L5, D7, D15 настраиваются как задачи/уведомления, не как email-шаблоны):

**Лиды (4 шаблона):**
- [ ] L1 — Автоответ при создании лида
- [ ] L3 — Не удалось связаться (первое)
- [ ] L4 — Не удалось связаться (повторное, кейс)
- [ ] L6 — Конвертация лида в сделку

**Сделки (13 шаблонов):**
- [ ] D1 — Welcome сделка (без лида)
- [ ] D2 — Образцы отправлены
- [ ] D3 — Обратная связь по образцам
- [ ] D4 — КП согласовано
- [ ] D5 — Счёт выставлен
- [ ] D6 — Напоминание об оплате
- [ ] D8 — Заказ произведён
- [ ] D9 — Оплата получена
- [ ] D10 — NPS + закрытие
- [ ] D11 — Отказ: цена
- [ ] D12 — Отказ: продукт
- [ ] D13 — Отказ: конкурент
- [ ] D14 — Отказ: ассортимент

## Шаг 3. Настроить роботов в воронке ЛИДОВ

### 3.1. Откройте настройку роботов

`CRM` → `Лиды` → вверху нажмите **«Роботы»** (или «Автоматизация»)

Откроется визуальная карта стадий лидов с возможностью добавлять роботов на каждую стадию.

---

### Стадия: «Не обработан»

#### Робот L1 — Автоответ клиенту (через 5 минут)

1. Нажмите **«+»** на стадии «Не обработан»
2. Выберите **«Отправить email клиенту»** (раздел «Коммуникация с клиентом»)
3. Заполните:
   - **Название робота:** `L1 — Автоответ`
   - **От кого:** выберите общий ящик (например, sales@verdetech.ru)
   - **Кому:** Email лида
   - **Тема:** скопируйте из шаблона L1 (или выберите шаблон)
   - **Текст:** выберите шаблон L1 из списка
4. **Задержка:** нажмите «Добавить условие» → «После предыдущего робота» → **5 минут**
5. Сохраните

#### Робот L2 — Пинг менеджеру (через 2 часа)

1. Нажмите **«+»** на стадии «Не обработан»
2. Выберите **«Уведомить сотрудника»** (раздел «Уведомления сотрудников»)
3. Заполните:
   - **Название:** `L2 — Лид не обработан 2 часа`
   - **Кому:** Ответственный
   - **Текст:** `Лид #LEAD.COMPANY_TITLE# (#LEAD.NAME#) не обработан более 2 часов. Свяжитесь с клиентом или передайте коллеге.`
4. **Задержка:** 2 часа
5. **Условие:** Только если лид всё ещё в статусе «Не обработан» (добавьте условие: Стадия = Не обработан)
6. Сохраните

---

### Стадия: «Не удалось связаться»

#### Робот L3 — Email клиенту (сразу)

1. **«+»** на стадии «Не удалось связаться»
2. **«Отправить email клиенту»**
3. Шаблон: **L3**
4. Задержка: **сразу** (0 минут)
5. Сохраните

#### Робот L4 — Повторный email с кейсом (через 3 дня)

1. **«+»** на той же стадии
2. **«Отправить email клиенту»**
3. Шаблон: **L4**
4. Задержка: **3 дня**
5. **Условие:** Стадия = «Не удалось связаться» (чтобы не отправлять если лид уже перешёл в другую стадию)
6. Сохраните

#### Робот L5 — Эскалация менеджеру (через 7 дней)

1. **«+»** на той же стадии
2. **«Уведомить сотрудника»**
3. Кому: Ответственный
4. Текст: `Лид #LEAD.COMPANY_TITLE# без связи 7 дней. Два письма отправлены, ответа нет. Примите решение — закрыть или попробовать альтернативные каналы.`
5. Задержка: **7 дней**
6. **Условие:** Стадия = «Не удалось связаться»
7. Сохраните

---

### Стадия: «Сделка создана» (конвертация)

#### Робот L6 — Email клиенту при конвертации

1. **«+»** на стадии конвертации (или настройте через **Триггеры** → «При конвертации лида»)
2. **«Отправить email клиенту»**
3. Шаблон: **L6**
4. Задержка: **сразу**
5. Сохраните

---

## Шаг 4. Настроить роботов в воронке СДЕЛОК

### 4.1. Откройте настройку роботов

`CRM` → `Сделки` → вверху нажмите **«Роботы»** (или «Автоматизация»)

---

### Стадия 1: «Связь установлена, потребность выявлена»

#### Робот D1 — Welcome (только для сделок без лида)

1. **«+»** на стадии 1
2. **«Отправить email клиенту»**
3. Шаблон: **D1**
4. Задержка: **сразу**
5. **Условие:** Добавьте условие → поле «Лид» → «не заполнено» (или «Источник» = не из конвертации лида)
   > Это ключевая настройка — D1 отправляется ТОЛЬКО если сделка создана напрямую, без лида. Иначе клиент уже получил L6.
6. Сохраните

#### Робот D2 — Образцы отправлены (по чекбоксу)

1. Этот робот настраивается НЕ на стадию, а **на изменение поля**
2. Перейдите: `CRM` → `Настройки` → `Автоматизация` → `Триггеры` (или в верхней панели роботов → вкладка «Триггеры»)
3. Добавьте триггер: **«Изменение поля»**
   - Поле: «Образцы отправлены»
   - Значение: «Да»
4. Действие: **«Отправить email клиенту»**
5. Шаблон: **D2**
6. Сохраните

> **Альтернативный вариант:** Если триггер по полю сложно настроить — добавьте робот D2 на стадию 1 с условием: поле «Образцы отправлены» = Да. Тогда менеджер сначала ставит чекбокс, потом вручную «обновляет» стадию.

#### Робот D15 — Сделка зависла 14 дней

1. **«+»** на стадии 1
2. **«Уведомить сотрудника»**
3. Текст: `Сделка #DEAL.TITLE# (#COMPANY.TITLE#) без движения 14 дней. Стадия: #DEAL.STAGE#. Свяжитесь с клиентом или переведите в отказ.`
4. Задержка: **14 дней**
5. **Условие:** Стадия = текущая (чтобы не сработало если сделка уже ушла дальше)
6. Сохраните

> **Скопируйте робот D15** на стадии 2, 3 и 4 — он одинаковый для всех ранних стадий.

## Шаг 5. Тестирование

### 5.1. Создайте тестовый лид

1. `CRM` → `Лиды` → **«Создать»**
2. Заполните:
   - Имя: `Тест Технолог`
   - Компания: `Тестовый завод`
   - Email: **ваш рабочий email** (чтобы получать письма)
   - Телефон: ваш телефон
3. Сохраните

### 5.2. Проверьте L1

- [ ] Через 5 минут на ваш email должно прийти письмо L1
- [ ] Проверьте: подставились ли переменные (имя, компания)?
- [ ] Проверьте: тема, прехедер, тело — всё как в шаблоне?

### 5.3. Проверьте L2

- [ ] Подождите 2 часа (или временно уменьшите задержку до 5 минут для теста)
- [ ] Менеджер должен получить уведомление L2

### 5.4. Переведите лид в «Не удалось связаться»

- [ ] Должно прийти L3 — сразу
- [ ] Через 3 дня (или уменьшите задержку) — L4 с кейсом
- [ ] Через 7 дней — L5 менеджеру

### 5.5. Конвертируйте лид в сделку

1. В карточке лида нажмите **«Создать на основании»** → **«Сделка»**
2. [ ] Должно прийти L6

### 5.6. Пройдите воронку сделки

Переводите тестовую сделку по стадиям и проверяйте:

- [ ] Стадия 1 → D1 (только если создана без лида — для теста создайте отдельную сделку напрямую)
- [ ] Поставьте чекбокс «Образцы отправлены» → D2
- [ ] Стадия 2 → D3
- [ ] Стадия 3 → D4
- [ ] Стадия 4 → D5
- [ ] Подождите (или уменьшите задержку) → D6 через 3 дня
- [ ] Стадия 5 → D8
- [ ] Стадия 6 → D9
- [ ] Стадия 7 → D10 через 1 день

### 5.7. Проверьте отказы

Создайте 4 отдельные тестовые сделки и переведите каждую в соответствующий отказ:

- [ ] Стадия 8 → D11 (цена)
- [ ] Стадия 9 → D12 (продукт)
- [ ] Стадия 10 → D13 (конкурент)
- [ ] Стадия 11 → D14 (ассортимент)

### 5.8. Чек-лист после тестирования

- [ ] Все переменные подставляются корректно
- [ ] Письма приходят от правильного адреса (sales@verdetech.ru)
- [ ] Задержки срабатывают (5 мин, 2 ч, 3 дня, 7 дней, 14 дней)
- [ ] Условия работают (D1 не приходит при конвертации из лида, D6 не приходит если уже оплатили)
- [ ] Уведомления менеджеру приходят (L2, L5, D7, D15)
- [ ] Прехедеры отображаются в почтовом клиенте

### 5.9. После успешного тестирования

1. **Удалите тестовые лиды и сделки** (или пометьте их)
2. **Верните задержки** к боевым значениям (если уменьшали для теста)
3. **Уведомьте менеджеров**, что автоматизация запущена — покажите им какие письма уходят и когда

## Частые проблемы

| Проблема | Решение |
|---|---|
| Переменные не подставляются (видно `#LEAD.NAME#` в письме) | Проверьте, что используете кнопку `#` в редакторе, а не вбиваете макросы вручную |
| Письмо не уходит | Проверьте: подключён ли почтовый ящик в CRM → Настройки → Почтовые ящики |
| Робот не срабатывает | Проверьте условия и задержки. Убедитесь, что робот активирован (переключатель ВКЛ) |
| D1 приходит вместе с L6 | Проверьте условие в D1: поле «Лид» должно быть пустым |
| D6 приходит после оплаты | Проверьте условие: стадия должна быть «Счёт выставлен» |
| Уведомления менеджеру не приходят | Проверьте настройки уведомлений сотрудника в профиле Б24 |
